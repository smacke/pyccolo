# pyccolo JupyterLite demo

This directory builds the in-browser [JupyterLite](https://jupyterlite.readthedocs.io/)
(Pyodide) demo site for pyccolo — the target of the "launch in JupyterLite" badge in the
top-level README. It lets anyone try pyccolo's tracers, syntax augmentations, and source
transforms from a browser with nothing installed.

## Layout

- `build.sh` — builds the site into `./dist`. It builds the pyccolo wheel from *this
  checkout* (so the demo always reflects current source) and downloads the pure-Python
  runtime deps (`traitlets`, `typing_extensions`, plus `comm` for the Pyodide Jupyter
  stack) into `pypi/`. `jupyter lite build` auto-discovers every wheel in that
  `<lite-dir>/pypi/` directory, copies them into the site, and merges them into the site's
  `pypi/all.json` index — so
  [piplite](https://jupyterlite.readthedocs.io/en/latest/howto/pyodide/wheels.html)
  resolves `%pip install pyccolo` from the bundled wheel offline (it checks the local index
  before falling back to PyPI).
- `content/demo.ipynb` — the landing notebook shown by the badge link.
- `pypi/`, `.jupyterlite.doit.db` — generated by `build.sh` (git-ignored).

## Build & serve locally

```bash
make jupyterlite         # build into ./dist  (needs network: downloads Pyodide + wheels)
make jupyterlite-serve   # build if needed, then serve at http://127.0.0.1:8000
make jupyterlite-dev     # build + serve in one step
```

Then open <http://127.0.0.1:8000/lab/index.html?path=demo.ipynb>.

## Deploy

Deployment to GitHub Pages is **manual** via the `Deploy JupyterLite demo` workflow
(`.github/workflows/deploy-jupyterlite.yml`) — run it from the Actions tab or with
`gh workflow run "Deploy JupyterLite demo"`. It requires the repository's Pages source to be
set to **GitHub Actions** (Settings → Pages).
